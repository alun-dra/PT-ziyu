1. Descripción del sistema

Ziyu Jardinería es una aplicación web construida con Django 6.0 que permite a una empresa de jardinería gestionar las solicitudes de visita de sus clientes. El sistema implementa el patrón MVT (Model-View-Template) de Django sin usar Django REST Framework ni React.

Tecnologías utilizadas

Backend Django 6.0.6, Python 3.12+
Base de datos PostgreSQL 14+ con psycopg 3
Frontend HTML5, CSS3, JavaScript vanilla
Mapas Leaflet.js + OpenStreetMap
Geocodificación Nominatim (reverse geocoding)
Herramientas dev django-extensions (shell_plus)

2. Comandos de instalación

Prerrequisitos

  • Python 3.12 o superior
  • PostgreSQL 14 o superior
  • pip (gestor de paquetes de Python)

Pasos de instalación

1
Clonar el repositorio y entrar al directorio 
cd backend-ziyu
2
Crear y activar el entorno virtual 
# Windows:
python -m venv env
env\Scripts\activate

# Linux/Mac:
python3 -m venv env
source env/bin/activate
3
Instalar dependencias 
pip install -r requirements.txt
4
Configurar variables de entorno 
# Editar .env con tus credenciales:
SECRET_KEY=tu-clave-secreta-aqui
DB_NAME=ziyu
DB_USER=postgres
DB_PASSWORD=tu-password
DB_HOST=localhost
DB_PORT=5432
5
Crear la base de datos en PostgreSQL 
psql -U postgres -c "CREATE DATABASE ziyu;"
6
Ejecutar migraciones 
python manage.py migrate
7
Crear superusuario 
python manage.py createsuperuser
8
Cargar datos iniciales 
python manage.py load_initial_data

Crea 5 tipos de servicio y 3 jardineros de ejemplo.

9
Iniciar el servidor de desarrollo 
python manage.py runserver

Accede a http://localhost:8000

Ejecutar tests

python manage.py test visits -v 2

3. Funcionalidad de la aplicación

Público

Formulario de solicitud de visita

Los clientes completan un formulario con datos de contacto, dirección, tipo de servicio, metros cuadrados del jardín y horario preferido. Incluye un mapa interactivo Leaflet que al hacer clic actualiza automáticamente las coordenadas y la dirección mediante geocodificación inversa (Nominatim).

GET / — Muestra el formulario
POST / — Crea la solicitud con estado "pendiente"
Protegido

Panel de empresa (Dashboard)

Vista interna protegida con @login_required. Muestra tarjetas con estadísticas (total, pendientes, asignadas, confirmadas), filtros por estado y tipo de servicio, tabla responsiva con todas las solicitudes, y un mapa con las ubicaciones marcadas. Permite asignar jardineros mediante un modal.

GET /panel/ — Dashboard con filtros opcionales por query params
Protegido

Asignación de jardinero

Vista POST-only protegida. Valida que no haya solapamiento de horarios antes de asignar. Si el jardinero ya tiene una visita en la misma fecha y horario superpuesto, la asignación se rechaza con un mensaje de error.

POST /asignar/<id>/ — Asigna jardinero, cambia estado a "asignada"
Público

Confirmación y cancelación del cliente

El cliente accede mediante un token UUID único (previene IDOR). Puede ver los detalles de su visita, confirmar la cita si tiene jardinero asignado, o cancelarla. Los estados se actualizan en tiempo real.

GET /confirmar/<uuid>/ — Ver detalles de la visita
POST /confirmar/<uuid>/ — Confirmar la visita
POST /cancelar/<uuid>/ — Cancelar la visita
Protegido

Admin de Django

Panel de administración personalizado con list_display, filtros, búsqueda, fieldsets y date_hierarchy. Permite editar estados y jardineros directamente en la lista.

/admin/ — Gestión completa de modelos

7. Seguridad

🔒
CSRF

Todos los formularios POST incluyen {% templatetag openblock %} csrf_token {% templatetag closeblock %}. El middleware de Django valida el token en cada petición.

🔑
IDOR Prevention

Las URLs de confirmación usan UUID tokens aleatorios en lugar de IDs secuenciales. Un atacante no puede adivinar el token de otro usuario.

🔐
Autenticación

Panel de empresa y documentación protegidos con @login_required. Asignación usa @require_POST.

🛡
XSS Prevention

Auto-escaping de Django en templates. Función escapeHtml() en JavaScript para popups del mapa. Sin uso de mark_safe.

🛠
SQL Injection

Uso exclusivo del ORM de Django. Ninguna consulta SQL cruda en el código.

✅
Validación

Validación server-side en Forms y Models. Fechas futuras, rangos horarios, solapamiento de jardineros.

8. Operaciones con shell_plus

Iniciar shell_plus (auto-importa todos los modelos):

python manage.py shell_plus

Ejemplos de uso

# Ver todas las solicitudes
VisitRequest.objects.all()

# Filtrar por estado
VisitRequest.objects.filter(status='pendiente')

# Buscar por nombre de cliente
VisitRequest.objects.filter(client_name__icontains='María')

# Eliminar una solicitud específica por ID
solicitud = VisitRequest.objects.get(pk=1)
print(f"Eliminando: {solicitud}")
solicitud.delete()

# Eliminar todas las solicitudes canceladas
eliminadas, _ = VisitRequest.objects.filter(status='cancelada').delete()
print(f"Se eliminaron {eliminadas} solicitudes canceladas")

# Ver solicitudes de un jardinero
VisitRequest.objects.filter(gardener__first_name='Carlos')

# Verificar que se eliminó
VisitRequest.objects.filter(pk=1).exists()  # False
